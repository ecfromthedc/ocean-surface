Side panel → agent context

The browser is not a place Ocean visits. It is where Ocean lives.

The Chrome extension should not feel like a chat box bolted onto the web. It should be a cockpit: attached to the active tab, aware of the selection, fluent in the page, and wired directly into the same local-first agent brain.

Current reality

The side panel works. The context pipe does not.

The extension is already a real MV3 side-panel shell. The missing part is not UI. It is identity and page context in the agent request.

01

Extension shell

extension/manifest.json, background.js, sidepanel.html, and sidepanel.js package the same Leptos/WASM Surface app.

02

Daemon connection

In chrome-extension:// mode, Surface skips /api/config and talks directly to 127.0.0.1:4780.

03

Blind turn

The request still says client_type: surface-web and carries no active tab, URL, title, selection, or page excerpt.
